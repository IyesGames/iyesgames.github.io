Expand description

Abstract interface through which the Host communicates with the Game

The “Game” is an implementation of the mechanics for a specific game mode. For example: the multiplayer MineWars game (proprietary) or the simplified singleplayer minesweeper mode (open-source).

type OutEvent: Clone + Send + Sync + 'static

Anything that happened that needs to be communicated to a player

When the Host calls either Game::input_action or Game::unsched to drive the Game, it can call Host::msg to send output events. They will be broadcast to all player ids selected with Plids.

source

type SchedEvent: Eq + Hash + Send + Sync + 'static

For things that need to be triggered on a timeout

Game code calls Host::sched, passing a value and time instant. The host code should store the value along with a timer. When the time instant has passed, host code calls Game::unsched, passing the value that was stored back to the game code.

source

type InputAction: Clone + Send + Sync + 'static

For things that are triggered by player input

Host code should call Game::input_action, passing an appropriate value, whenever the player wishes to perform an action in the game.

source

type InitData: Send + Sync + 'static

Data passed at initialization to set up the game.
