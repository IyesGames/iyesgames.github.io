The matrix for mapping from AxialPos, to world position when hexes are arranged in column format (“flat top” per Red Blob Games). See Size and Spacing at Red Blob Games for an interactive visual explanation, but note that: 1) we consider increasing-y to be the same as “going up”, while RBG considers increasing-y to be “going down”, 2) our vectors have magnitude 1 (in order to allow for easy scaling based on grid-size)
